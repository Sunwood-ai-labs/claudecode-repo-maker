# Claude Code - リポジトリ作成自動化ガイド

このドキュメントは、ユーザーの指示を元に新規GitHubリポジトリの作成、README作成、作業報告書の生成を自動化するためのガイドです。

## 🎯 目的

ユーザーからの簡単な指示で以下の作業を自動的に実行します：

1. GitHubリポジトリの作成
2. プロジェクト用のREADME.mdの作成
3. 作業報告書の生成

## 📋 前提条件

- 環境変数 `GITHUB_TOKEN` が設定されていること
- GitHub APIへのアクセス権限があること
- gitコマンドが利用可能であること

## 🚀 使用方法

### 基本的な指示形式

```
[プロジェクト名]のリポジトリを作成して
```

### 例

```
サイコロアプリのリポジトリを作成して
ToDoアプリのリポジトリを作成して
天気予報アプリのリポジトリを作成して
```

## 📝 実行される作業フロー

### ステップ1: GitHubリポジトリの作成

GitHub REST APIを使用してリポジトリを作成します。

**実行コマンド:**
```bash
curl -X POST -H "Authorization: token $GITHUB_TOKEN" \
  -H "Accept: application/vnd.github.v3+json" \
  https://api.github.com/user/repos \
  -d '{
    "name": "リポジトリ名",
    "description": "プロジェクトの説明",
    "private": false,
    "auto_init": false
  }'
```

**出力情報:**
- リポジトリURL
- リポジトリID
- 作成日時
- デフォルトブランチ

### ステップ2: README.mdの作成

プロジェクトに適したREADME.mdを自動生成します。

**README.mdの標準構成:**

```markdown
# プロジェクト名

プロジェクトの簡潔な説明

## 📝 概要
詳細な説明

## ✨ 機能
- 機能1
- 機能2
- 機能3

## 🛠️ 技術スタック
使用する技術の一覧

## 🚀 インストール方法
セットアップ手順

## 💻 開発
開発環境のセットアップ

## 🎯 使用例
具体的な使用例

## 🤝 コントリビューション
貢献方法

## 📄 ライセンス
ライセンス情報

## 👤 作成者
作成者情報
```

**実行プロセス:**
1. リポジトリをクローン
2. README.mdを作成
3. Gitコミット
4. リモートリポジトリにプッシュ

```bash
git clone https://github.com/[organization]/[repo-name].git
cd [repo-name]
# README.mdを作成
git add README.md
git commit -m "docs: 初期README作成"
git push -u origin main
```

### ステップ3: 作業報告書の生成

作業内容を詳細に記録した報告書を生成します。

**報告書の保存場所:**
```
claudecode-repo-maker/reports/work-report-YYYY-MM-DD.md
```

**報告書の標準構成:**

```markdown
# 作業報告書 - [プロジェクト名]

**作成日:** YYYY年MM月DD日
**作業者:** Claude Code
**プロジェクト:** [プロジェクト名]

## 📋 作業概要
実施した作業の概要

## ✅ 実施内容

### 1. GitHubリポジトリの作成
- リポジトリ情報
- 実行コマンド
- 結果

### 2. README.mdの作成
- README構成
- 作成内容

### 3. Gitコミット＆プッシュ
- コミット情報
- 変更内容

## 📊 作業結果サマリー
統計情報

## 🔗 関連リンク
リポジトリURLなど

## 📝 備考
技術的な注意点や次のステップ
```

## 🔧 カスタマイズ可能な項目

### リポジトリ設定

- **リポジトリ名:** プロジェクトに基づいて自動生成
- **説明:** プロジェクトの内容から自動生成
- **公開設定:** デフォルトはパブリック（変更可能）
- **Organization:** Sunwood-ai-labs（変更可能）

### README.md

- **技術スタック:** プロジェクトに応じて調整
- **機能一覧:** プロジェクトの要件に基づいて生成
- **セクション:** 必要に応じて追加・削除可能

### 報告書

- **フォーマット:** マークダウン形式
- **詳細レベル:** 包括的な情報を含む
- **保存場所:** `reports/` ディレクトリ

## 🛡️ エラーハンドリング

### GitHub API エラー

**対処方法:**
- トークンの有効性確認
- レート制限の確認
- リトライロジックの実装（指数バックオフ）

### Git操作エラー

**対処方法:**
- 認証情報の確認
- リモートURLの修正
- コミット署名の無効化（必要に応じて）

### ネットワークエラー

**対処方法:**
- 最大4回のリトライ（2秒、4秒、8秒、16秒の指数バックオフ）
- タイムアウト設定（30秒）

## 🔄 ワークフロー例

### 完全な実行例

**ユーザー指示:**
```
電卓アプリのリポジトリを作成して
```

**実行される処理:**

1. **リポジトリ作成**
   ```
   リポジトリ名: calculator-app
   説明: シンプルな電卓アプリケーション
   URL: https://github.com/Sunwood-ai-labs/calculator-app
   ```

2. **README.md作成**
   - プロジェクト概要
   - 四則演算機能
   - HTML/CSS/JavaScript技術スタック
   - インストール・使用方法

3. **作業報告書生成**
   ```
   reports/work-report-2025-11-15.md
   - リポジトリ作成詳細
   - README構成
   - 次のステップ提案
   ```

4. **Gitコミット＆プッシュ**
   ```
   コミット: "docs: 電卓アプリ用のREADMEを作成"
   プッシュ先: main ブランチ
   ```
